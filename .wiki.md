# Project Summary
The project is an AI-powered news portal that automates the fetching, rewriting, and publishing of real-time news articles. It provides a user-friendly interface for writers to manage content, review source articles, and publish across various categories. Recent improvements include a refined TinyMCE rich text editor with direct file uploads and enhanced media handling. The portal aims to streamline the news publishing process, making it efficient and accessible while leveraging advanced AI technologies for content creation and analysis.

# Project Module Description
- **Admin Panel**: Comprehensive content management interface for article editing, organized metadata layout, and enhanced TinyMCE editor.
- **Blog Section**: Displays SEO-optimized articles on AI-powered news curation and publishing.
- **Real-Time News Fetching**: Utilizes the Apify Google News Scraper API to fetch and rewrite articles, now enhanced with an AI fallback for failed scrapes, including handling HTTP errors such as 403.
- **Image and Video Management**: Writers can upload and manage images and videos with auto-resizing capabilities through the TinyMCE editor.
- **Web Scraper**: Supports unlimited URLs for scraping, integrates AI rewriting, and allows previewing before publishing. Now includes a "Words Length" dropdown feature and a robust multi-layer fallback system for scraping failures.
- **Manual Submission**: Users can manually create articles with an improved layout and organized metadata fields featuring the TinyMCE editor.
- **Search Functionality**: Enhanced capabilities for finding articles based on various attributes.
- **Newsletter Engine**: Manages newsletter operations, including subscriber management, template selection, newsletter composition, and analytics.

# File Description Inventory
- **app/backend/main.py**: Entry point for the backend server.
- **app/backend/services/newsletter_service.py**: Service for managing newsletters, including subscriber management, queue processing, and analytics.
- **app/backend/routers/newsletter_engine.py**: API routes for newsletter operations, including subscription, composing newsletters, and A/B testing.
- **app/backend/schemas/newsletter.py**: Pydantic schemas for validating requests and responses related to newsletters.
- **app/backend/models/newsletter_queue.py**: Database model for managing queued newsletters.
- **app/backend/models/newsletter_sent_history.py**: Database model for tracking sent newsletters.
- **app/backend/models/newsletter_templates.py**: Database model for storing newsletter templates.
- **app/frontend/pages/Newsletter.tsx**: Main component for the Newsletter Engine, including dashboard, subscribers, templates, and composition functionalities.
- **app/frontend/components/SubscribeWidget.tsx**: Public subscribe widget for the homepage.
- **app/frontend/components/RichTextEditor.tsx**: Enhanced TinyMCE wrapper for rich text editing, now supporting direct image and video uploads with size measurement and responsive design.
- **app/frontend/pages/ArticleDetail.tsx**: Displays published articles with responsive image and video handling.
- **app/frontend/src/lib/auth.ts**: Updated authentication client with retry logic for transient errors.
- **app/frontend/src/lib/api.ts**: Resilient SDK client wrapper with automatic retry logic for transient errors.
- **app/frontend/src/lib/retry.ts**: Centralized retry utility for handling transient DNS/balancer resolve errors with exponential backoff.
- **app/backend/services/news_service.py**: Service for managing news scraping and rewriting functionalities, now with a multi-layer fallback for HTTP errors.
- **app/backend/routers/news.py**: API routes for news fetching and rewriting operations.
- **app/backend/schemas/news.py**: Pydantic schemas for validating requests and responses related to news scraping.

# Technology Stack
- **Frontend**: React, TypeScript, Vite
- **Backend**: Python, FastAPI
- **Database**: SQLAlchemy with a relational database
- **AI Technology**: DeepSeek for article rewriting and Winston AI for content analysis
- **Web Scraping**: BeautifulSoup, httpx for scraping
- **Image and Video Storage**: Atoms Cloud for uploads
- **Rich Text Editing**: TinyMCE (self-hosted, free tier)
- **Deployment**: Vite Prerender Plugin for SEO optimization

# Usage
1. **Install Dependencies**: Run `pnpm install` in both the `app/backend` and `app/frontend` directories.
2. **Build**: Use `pnpm run build` to compile the project.
3. **Run**: Start the backend server and frontend application using their respective commands.
